# UI Review: Ремонт духовых шкафов — oven-repair

**Сайт:** https://fedosis.github.io/landings/oven-repair/  
**Дата ревью:** Июнь 2026  
**Версия:** v4 (текущая)

---

## 🔴 Критические проблемы (требуют немедленного исправления)

### 1. Сломанная CSS-вложенность `.pricing-card`

**Где:** `index.html`, строки 727–813  
**Проблема:** Внутри `.pricing-card { }` открыт ещё один `.pricing-card { }` без префикса `&`. В стандарте CSS Nesting это эквивалентно селектору `.pricing-card .pricing-card` (потомок), а не самому элементу. Все базовые стили карточки (фон, бордер, паддинг, высота, flex-direction) **не применяются** к элементу `.pricing-card`.

```css
/* ⚠️ ТАК СЕЙЧАС */
.pricing-card {
  .pricing-card {           /* = .pricing-card .pricing-card (descendant!) */
    background: ...;
    border: ...;
    border-radius: ...;
    padding: 36px 28px;
    ...
  }
}

/* ✅ ДОЛЖНО БЫТЬ */
.pricing-card {
  background: ...;
  border: ...;
  border-radius: ...;
  padding: 36px 28px;
  height: 100%;
  ...
}
.pricing-card::before { ... }
.pricing-card:hover { ... }
.pricing-card .price-icon { ... }
```

### 2. Иконка бургер-меню не видна на белом фоне при скролле

**Где:** строка 1711  
**Проблема:** `<i class="bi bi-list text-white">` — жёсткий белый цвет. Когда навбар получает класс `.scrolled`, его фон становится `rgba(255,255,255,0.92)`. Белая иконка на белом фоне исчезает.  

**Исправление:** Добавить CSS:
```css
.navbar.scrolled .navbar-toggler i {
  color: var(--text-primary) !important;
}
```

### 3. Плейсхолдеры телефона и email по всему сайту

**Где:** Schema.org (строки 1677–1678), навбар (строка 1721), контакты (строка 2567–2570), футер (строка 2638–2639)  
**Проблема:** Везде `+7XXX`, `info@example.com`, `+7-XXX-XXX-XX-XX`. Реальные контакты не подставлены. Пользователь, который дойдёт до кнопки «Позвонить» — не сможет позвонить.  
**Исправление:** Заменить все плейсхолдеры на реальный номер телефона и email.

### 4. WhatsApp и Viber в контактах

**Где:** строка 2568 (контактная форма), строки 2642–2644 (футер)  
**Проблема:** В контактах есть WhatsApp и Viber, а по требованиям пользователя должен быть только Telegram.  
**Исправление:** Удалить все ссылки на WhatsApp, Viber и другие мессенджеры, оставить только Telegram.

### 5. Пустое `image` в Schema.org и отсутствие `og:image`

**Где:** Schema.org строка 1680 (`"image": ""`), мета-теги (строки 11–16)  
**Проблема:** При шеринге в соцсетях нет картинки-превью. Schema.org не содержит ссылки на изображение.  
**Исправление:** Добавить `<meta property="og:image" content="https://fedosis.github.io/landings/oven-repair/assets/logo.png">` и `"image": "https://fedosis.github.io/landings/oven-repair/assets/logo.png"` в Schema.org.

## 🟡 Средние проблемы

### 6. Пустая колонка `col-lg-5` в hero

**Где:** строка 1769–1771  
**Проблема:** На десктопе справа пустует ~41% ширины экрана. Фото мастера висит фоном всего hero, а колонка для него пустая.  
**Исправление:** Переместить фото мастера из `.hero-bg` в эту колонку, либо убрать колонку и сделать контент на всю ширину.

### 7. Hero-фон загружается с `loading="lazy"`

**Где:** строка 1735  
**Проблема:** Изображение hero — самый критичный визуальный элемент выше сгиба. `loading="lazy"` откладывает его загрузку, что вредит LCP (Largest Contentful Paint).  
**Исправление:** Заменить на `fetchpriority="high"`, убрать `loading="lazy"`.

### 8. Тяжёлый PNG для водяного знака

**Где:** строка 1359 (`background: url('assets/logo.png')`)  
**Проблема:** `logo.png` весит 267 КБ и отображается с opacity 0.04. Это пустая трата трафика.  
**Исправление:** Конвертировать в WebP или использовать `logo.jpg` (172 КБ).

### 9. Два бренда без логотипов

**Где:** строки 2175, 2178  
**Проблема:** KUPPERSBERG и MAUNFELD отображаются текстом, в то время как все остальные бренды — с изображениями. Визуально не единообразно.  
**Исправление:** Добавить PNG-логотипы для этих брендов или перевести все бренды на текстовый формат.

### 10. Фото в разделе «О мастере» — не портрет мастера

**Где:** строка 2513 (`src="assets/repair-1.jpg"`)  
**Проблема:** Используется фото ремонта, а не портрет мастера. В assets уже есть `master-photo.jpg`.  
**Исправление:** Заменить `repair-1.jpg` на `master-photo.jpg`.

### 11. Карта — плейсхолдер вместо настоящей карты

**Где:** строки 2590–2594  
**Проблема:** Вместо карты района обслуживания — заглушка с иконкой геолокации.  
**Исправление:** Вставить Yandex Maps или 2GIS через `<iframe>`.

### 12. Нет фокус-ловушки в лайтбоксе

**Где:** `.lightbox` (строки 922–976 CSS, 2264–2269 HTML)  
**Проблема:** При открытом лайтбоксе клавиша Tab может увести фокус под затемнение. Нет обработки `inert` на фоне.  
**Исправление:** Добавить фокус-ловушку с `focusin`/`focusout` или установить `inert` на остальные элементы при открытии лайтбокса.

## 🔵 Мелкие замечания

### 13. Галерея работ — нечётное количество (7 шт.)

**Где:** `.works-grid` (строки 2195–2259)  
**Проблема:** При auto-fill/grid последний элемент висит один в ряду на десктопе.  
**Исправление:** Добавить 8-ю работу или использовать `repeat(4, 1fr)` на десктопе.

### 14. Утечка интервала в автоплее отзывов

**Где:** JavaScript, строки 2734–2741  
**Проблема:** При `mouseleave` не чистится старый `interval` перед созданием нового. После нескольких наведений мыши запускается несколько параллельных интервалов.  
**Исправление:** Сохранять `intervalId` во внешней переменной и делать `clearInterval` перед каждым новым `setInterval`.

### 15. Нет `aria-label` у CTA-кнопки в навбаре

**Где:** строка 1721  
**Проблема:** Кнопка «Позвонить» в навигации не имеет aria-label для скринридеров.  
**Исправление:** Добавить `aria-label="Позвонить мастеру"`.

### 16. Форма с `novalidate` — нет браузерной валидации

**Где:** строка 2544  
**Проблема:** Отключена стандартная HTML5-валидация, есть только кастомная проверка после submit. Пользователь не видит подсказок при вводе.  
**Исправление:** Убрать `novalidate` или добавить класс `was-validated` Bootstrap для визуальной обратной связи.

### 17. Мёртвый CSS — класс `.photo-frame`

**Где:** строки 441–478  
**Проблема:** Полностью описанный в CSS класс `.photo-frame` нигде не используется в HTML. Это legacy-код.  
**Исправление:** Удалить из CSS или использовать для фото мастера.

### 18. Watermark на hero — почти незаметен

**Где:** строка 1733 (`<section class="hero watermark">`)  
**Проблема:** На тёмном фоне hero watermark с 4% opacity практически не виден. На светлых секциях он заметен, на hero — нет.  
**Исправление:** Либо убрать `watermark` с hero, либо оставить (некритично).
